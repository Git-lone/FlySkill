# 问题模板库

本文档提供了各类需求澄清问题的详细模板，供 Skill 智能选择使用。

## 功能需求类问题

### 核心目标
- 这个功能的核心目标是什么？要解决什么问题？
- 这个功能的成功标准是什么？如何衡量是否达到目标？
- 这个功能是为了解决现有问题，还是增加新能力？

### 用户相关
- 目标用户是谁？（内部员工 / 外部客户 / 管理员 / 其他）
- 用户的技术水平如何？（技术专家 / 普通用户 / 新手）
- 预期有多少用户会使用这个功能？

### 参考和优先级
- 是否有类似的参考实现或竞品功能可以参考？
- 这个功能的优先级如何？（P0 紧急 / P1 重要 / P2 普通）
- 是否有明确的上线时间要求？

## 技术约束类问题

### 技术栈
- 需要使用特定的技术栈或框架吗？（如 React、Vue、Django 等）
- 是否需要使用特定的编程语言？
- 是否有第三方库或工具的限制？

### 兼容性
- 是否需要兼容现有系统或接口？
- 需要支持哪些浏览器或设备？
- 是否需要考虑向后兼容性？

### 安全和合规
- 是否有安全要求？（如数据加密、身份认证）
- 是否有权限控制要求？（谁可以访问、谁可以修改）
- 是否有合规要求？（如 GDPR、等保等）

## 数据相关类问题

### 数据来源
- 数据来源是什么？（用户输入 / API / 数据库 / 文件上传 / 第三方服务）
- 数据是实时的还是批量的？
- 数据的更新频率是多少？

### 数据格式
- 预期的数据格式是什么？（JSON / CSV / XML / Excel / 其他）
- 数据结构是什么样的？（字段、类型、必填项）
- 是否需要数据验证？验证规则是什么？

### 数据规模
- 预期的数据量级是多少？（影响技术选型和性能优化）
- 单次处理的数据量有多大？
- 数据增长速度如何？

### 数据存储
- 数据是否需要持久化？
- 如果需要，存储在哪里？（数据库 / 文件系统 / 云存储）
- 数据需要保留多久？是否有归档要求？

## 用户体验类问题

### 交互方式
- 期望的交互方式是什么？（表单 / 弹窗 / 页面跳转 / 拖拽 / 其他）
- 用户操作的主要流程是什么？
- 是否需要提供快捷操作或批量操作？

### 界面设计
- 是否有界面风格或设计规范要求？
- 是否有品牌色或设计系统？
- 是否需要响应式设计？

### 设备支持
- 是否需要支持移动端？（手机 / 平板）
- 主要使用场景是桌面端还是移动端？
- 是否需要支持触摸操作？

### 无障碍
- 是否需要考虑无障碍访问（Accessibility）？
- 是否需要支持屏幕阅读器？
- 是否需要支持键盘导航？

## 性能与规模类问题

### 响应时间
- 是否有响应时间要求？（如页面加载 < 2秒，API 响应 < 500ms）
- 用户可以接受的最长等待时间是多少？
- 是否需要显示加载进度？

### 并发和负载
- 预期的并发用户数是多少？
- 预期的峰值流量是多少？
- 是否需要考虑限流或降级策略？

### 扩展性
- 是否需要考虑扩展性？（未来可能的功能扩展）
- 系统是否需要支持水平扩展？
- 是否需要考虑模块化设计？

### 离线和缓存
- 是否需要支持离线使用？
- 是否需要缓存策略？
- 数据同步策略是什么？

## 业务逻辑类问题

### 业务规则
- 具体的业务规则是什么？
- 是否有特殊的计算逻辑？
- 是否有条件分支或例外情况？

### 工作流
- 完整的业务流程是什么？
- 是否需要审批流程？
- 是否需要通知或提醒？

### 集成对接
- 是否需要与其他系统集成？
- 第三方系统的接口文档在哪里？
- 是否需要处理集成失败的情况？

## 问题选择策略

根据需求的关键词，智能选择相关问题：

| 需求关键词 | 优先问题类型 |
|-----------|------------|
| "界面"、"页面"、"UI" | 用户体验类 + 功能需求类 |
| "性能"、"优化"、"慢" | 性能与规模类 + 数据相关类 |
| "数据"、"导入"、"导出" | 数据相关类 + 功能需求类 |
| "接口"、"API"、"集成" | 技术约束类 + 业务逻辑类 |
| "登录"、"权限"、"安全" | 技术约束类（安全） + 功能需求类 |
| "报表"、"统计"、"分析" | 数据相关类 + 功能需求类 |
| "流程"、"审批"、"工作流" | 业务逻辑类 + 功能需求类 |
