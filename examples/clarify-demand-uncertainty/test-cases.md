## 完整流程测试

### 测试场景：从模糊需求到实施

**步骤 1：提出模糊需求**
```
我想在应用里加一个搜索功能。
```

**预期：** Skill 触发，提出澄清问题

**步骤 2：回答问题**
```
- 搜索范围：用户列表
- 搜索字段：用户名、邮箱
- 搜索方式：实时搜索（输入时自动搜索）
- 位置：列表页顶部
- 使用现有的 Input 组件
```

**预期：** 生成需求文档 `docs/requirements-YYYYMMDD.md`

**步骤 3：确认需求**
```
确认，开始实施。
```

**预期：** Agent 根据需求文档开始实施

## 特殊情况测试

### 测试 7：用户回答"随便"

**测试输入：**
```
添加一个数据导出功能。
```

**Agent 提问：**
```
导出格式是什么？
- CSV
- Excel
- JSON
```

**用户回答：**
```
随便，都可以。
```

**预期行为：**
- ✅ Agent 主动推荐一个方案（如 CSV）
- ✅ 说明推荐理由
- ✅ 询问用户是否接受

---

### 测试 8：用户拒绝回答

**测试输入：**
```
优化一下列表页的性能。
```

**Agent 提问：**
```
预期的数据量是多少？
```

**用户回答：**
```
不知道，你看着办吧。
```

**预期行为：**
- ✅ Agent 使用行业最佳实践作为默认值
- ✅ 在需求文档中标注这是假设
- ✅ 继续其他问题或开始实施

---

### 测试 9：回答有冲突

**测试输入：**
```
添加一个实时聊天功能。
```

**用户回答：**
```
- 需要支持 10000 并发用户
- 不需要使用 WebSocket，用普通 HTTP 就行
```

**预期行为：**
- ✅ Agent 指出矛盾（10000 并发用户用 HTTP 轮询不现实）
- ✅ 请求用户澄清优先级或调整需求
- ✅ 推荐合理的技术方案

## 故障排除

### Skill 没有触发？

1. **检查安装路径**
   ```bash
   ls -la .claude/skills/clarify-demand-uncertainty/
   # 或
   ls -la ~/.claude/skills/clarify-demand-uncertainty/
   ```

2. **检查 SKILL.md 格式**
   - 确保有正确的 YAML front matter
   - 确保文件编码是 UTF-8

3. **重启 Agent**
   - 重启 Claude Code 或你的 AI Agent

### Skill 触发了但行为不对？

1. **检查 SKILL.md 内容**
   - 确保"何时使用"和"何时不使用"部分清晰
   - 确保工作流程描述完整

2. **查看 Agent 日志**
   - 观察 Agent 是否正确理解了 Skill 的指令
